# 🎫 Ticket TM-001 - Slow Computer Investigation

## Overview

Task Manager is a Windows administration tool used to monitor system performance, running processes, startup applications and resource utilization.

This lab demonstrates how to investigate a user complaint regarding poor system performance using Task Manager.

---

# Ticket Information

| Field | Value |

| Status | Completed |

---

# Reported Issue

User reports that the laptop is running slowly and applications take longer than expected to open.

---

# Objective

Investigate system performance using Task Manager and identify any processes or applications consuming excessive system resources.

---

# Investigation Steps

1. Open Task Manager.
2. Review running processes.
3. Identify high CPU usage.
4. Identify high memory usage.
5. Review disk activity.
6. Review network activity.
7. Examine startup applications.
8. Document findings.

# Lessons Learned

This lab improved my understanding of how Task Manager can be used to investigate performance-related issues in Windows.

Key lessons learned:

- Task Manager provides real-time visibility into system resource usage.
- High CPU utilization may indicate a resource-intensive application or background process.
- Memory usage can help identify applications consuming excessive RAM.
- Disk activity can be used to detect storage bottlenecks.
- Network activity can reveal applications generating significant traffic.
- Startup applications can affect system boot times and overall performance.
- Task Manager is one of the first tools used by IT Support Engineers when troubleshooting slow computers.

This lab strengthened my ability to analyze system performance and identify potential causes of performance degradation.
